in planning we start by looking into what equipment is already laying around, i already have an arduino nano not being used, and an OLED screen (with i2c) used for an old school assignment. using these i plan to first make the user interface, of course there will be no way of editing the information at the start so it will only be displaying data and possibly turning on an LED for the given amount of time

when i have that working i plan to get a knob to alow me to change the values without needing to recompile the program each time

finally to interact with the camera i will need to short the to contact on the aux jack connected to the camera, to do that i will need a digital switch, the type of which i have yet to decide on, but i will need to be active for a long period with using large amounts of energy, lastly it must be able to handle the voltage at which the camera is running at, which is currently unknown to me as i haven't measured it


development of the remote will happen using a few different tools

  • Scheme It by DigiKey

    link
    this is a free online tool for making schematics, it has a large library of components and is easy to use
    it also has integrated features for making a BOM and ordering the components
    this is a tool i have used before and am familiar with, though i am in the process of switching to fusion 360 for this part, though their component library still feels somewhat laking in comparison

  • microchip studio by microchip

    link
    this is the IDE i will be using for programming the arduino, it is made by microchip and is free to use
    it supports most, if not all, AVR micro controllers which is what most arduinos are based on
    i have used this IDE before and am familiar with it, though i would prefer using most of the other IDE and editors i know, but this one simply has a great integration with AVR ccontrollers that you don't find anywhere else.

  • DTU Ballerup component library

    here i will attempt to find as many of the necessary components as possible, this is a library of components that DTU Ballerup has made available for use by all students both for study relevant projects and personal projects
    while they mostly carry passive components, they do have a few active components as well, they also carry methods for getting larger/more expensive components, tho i don't plan to use that in this project
